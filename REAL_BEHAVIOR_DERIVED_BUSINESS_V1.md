# Real Behavior Derived Business V1

## Purpose

Create new business candidates only from the owner's real payment behavior.

Past rankings are ignored.

Existing candidates are not used as comparison targets.

Reference paid behaviors:

- ChatGPT
- Uber One
- LUUP
- internet connection
- server
- streaming

Common traits:

- hassle reduction
- result certainty
- immediate use
- recurring use

## Candidate Set

Evaluate only these five candidates:

1. AI cancellation agent
2. AI phone agent
3. AI reservation booking agent
4. AI life cost reduction execution
5. AI price monitoring and purchase agent

## Executive Judgment

The strongest candidate is:

```text
AI life cost reduction execution
```

Reason:

It aligns best with actual payment behavior:

- saves money
- removes repeated hassle
- produces measurable result
- can use success-fee pricing
- has recurring utility
- can start without large upfront cost if scoped narrowly

The weakest direct candidate is:

```text
AI phone agent
```

Reason:

The desire is strong, but legal, trust, voice quality, escalation, and business-process risk are high.

The best near-term strategy is not broad automation.

It is:

```text
start with narrow execution tasks where outcome is measurable and user approval gates are clear.
```

## 1. AI Cancellation Agent

## One-Sentence Offer

```text
Cancel annoying subscriptions, memberships, or services without making the user deal with forms, phone calls, or retention traps.
```

## Desire Intensity

```text
5 / 5
```

Users strongly dislike cancellation friction.

Pain is easy to understand:

- unwanted billing
- hard-to-find cancellation page
- phone-only cancellation
- retention screens
- forgotten subscriptions
- anxiety around being charged again

This matches real payment behavior because it removes repeated hassle.

## Result Certainty

```text
4 / 5
```

Strong if the service targets cancellation types with clear processes.

Weak if cancellation requires:

- identity verification
- phone conversation
- physical document
- legal authority
- customer-specific negotiation

The product must avoid promising cancellation where execution is uncertain.

## Profit Structure

Possible pricing:

- 500-1,500 JPY per successful cancellation
- 2,980 JPY/month for cancellation monitoring
- success fee equal to one month of saved cost
- business plan for SaaS/subscription cleanup

Best structure:

```text
free scan of subscriptions
-> pay only when cancellation is executed or savings are found
```

## AI-Led Operation

```text
4 / 5
```

AI can handle:

- cancellation path research
- email drafting
- form instructions
- support message generation
- reminder tracking
- evidence checklist
- user approval prompts

Human or user approval is still needed for:

- login
- identity verification
- final confirmation
- phone-only edge cases

## Build vs Buy

```text
3 / 5
```

ChatGPT can draft cancellation emails.

But a product can win if it offers:

- provider-specific cancellation steps
- saved tracking
- reminders before renewal
- success/failure status
- evidence storage
- prebuilt flows

## 150,000 JPY/Month Path

Possible paths:

```text
150 cancellations x 1,000 JPY
50 users x 2,980 JPY/month
10 small businesses x 15,000 JPY cleanup pack
```

Feasible if customer access is solved.

## 1,000,000 JPY/Month Path

Possible but harder as pure B2C.

Paths:

```text
336 users x 2,980 JPY/month
100 businesses x 10,000 JPY/month
success fee from recurring savings
```

Better as:

```text
subscription and cost cleanup utility
```

not just one-off cancellation.

## Risk

The hardest problems:

- authority to act
- privacy
- login credential handling
- support escalation
- provider-specific process changes
- phone-only cancellations

## Judgment

```text
GO / CAUTION
```

Good candidate if scoped narrowly:

```text
subscription cancellation and renewal reminder assistant
```

Do not start with full代理.

Start with:

```text
AI-prepared cancellation flow + user final approval
```

## 2. AI Phone Agent

## One-Sentence Offer

```text
Let AI make annoying phone calls for reservations, confirmations, cancellations, and simple inquiries.
```

## Desire Intensity

```text
5 / 5
```

Very strong.

Use cases:

- restaurant booking
- clinic confirmation
- service inquiry
- cancellation
- rescheduling
- availability check
- price confirmation

## Result Certainty

```text
2 / 5
```

- unexpected questions

- background noise

- legal/consent issues
- escalation

- business usage by volume
- per successful booking/cancellation

```text
3 / 5
```

AI voice can handle simple calls, but human fallback may be needed.

AI can handle:

- call script generation
- intent extraction
- confirmation summaries
- follow-up messages

Hard:

- live voice reliability
- trust
- ambiguous conversations
- escalation
- regulatory compliance

## Build vs Buy

```text
4 / 5
```

ChatGPT cannot make calls by itself.

The product has stronger Build vs Buy resistance if it actually executes calls.

## 150,000 JPY/Month Path

Weak unless B2B or high-volume niche.

Examples:


But support and reliability risk are high.

## 1,000,000 JPY/Month Path

- recurring B2B demand
- low failure rate

- salons
- restaurants
- service businesses
- property managers

## Risk

Biggest risks:

- voice AI quality
- legal/consent
- business trust
- failed calls
- user embarrassment
- exception handling

```text

5 / 5
```

```text
4 / 5
```

- detect duplicates
- identify cheaper alternatives
- generate cancellation/switching instructions
- draft support messages
- track savings

Human/user approval required for:

- final cancellation
- switching providers
- payment changes
- personal data handling

## Build vs Buy

```text
4 / 5
```

ChatGPT can give advice, but the product wins if it:

- tracks recurring costs
- stores cost history
- monitors renewals
- executes reminders
- provides provider-specific steps
- measures savings
- charges based on result
