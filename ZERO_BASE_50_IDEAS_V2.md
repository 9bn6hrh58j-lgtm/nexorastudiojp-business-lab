# Zero Base 50 Ideas V2

## Purpose

Reevaluate `ZERO_BASE_50_IDEAS_V1.md` using stricter AI-substitution criteria.

Important change:

```text
"Users want it" is not enough.
```

The key question is:

```text
Will the user simply open ChatGPT or Claude and finish the job there?
```

Revenue Risk Scanner receives no special treatment.

Top candidates receive no protection.

## Evaluation Axes

Each score is 1 to 5.

| Axis | 5 Means | 1 Means |
|---|---|---|
| Build vs Buy | User is more likely to buy than build/use generic AI | User can easily self-serve with generic AI |
| ChatGPT resistance | ChatGPT is not enough | ChatGPT can mostly replace it |
| Claude resistance | Claude is not enough | Claude can mostly replace it |
| AI-led operation | Can be operated mostly by AI | Human work is central |
| Desire intensity | User strongly wants outcome | Weak desire |
| Result certainty | Output reliably creates value | Value is uncertain |

Total maximum:

```text
30
```

## Executive Judgment

V1 overvalued generic generation products.

Many high-scoring V1 ideas fail the new test because:

- ChatGPT can generate the same draft
- Claude can produce high-quality long-form text
- users do not need a separate product
- paid conversion requires workflow, saved context, integrations, or distribution

The new winners are not "AI writes text" products.

They are products where generic AI is weaker because the product includes:

- workflow tracking
- structured escalation
- recurring business context
- risk-sensitive templates
- evidence organization
- integrations
- benchmarks
- status/history
- domain-specific output constraints

## Key Recalculation Changes

## 1. Generic Writing Products Dropped

These looked strong in V1:

- AI就活ES生成
- AI商品説明文生成
- AI店舗SNS投稿生成
- AI広告コピー生成
- AIプレスリリース生成
- AI旅行プランナー
- AI料理レシピ生成

But in V2 they fall because:

```text
User can open ChatGPT or Claude and get a usable answer immediately.
```

Desire is real.

Willingness to pay for a separate product is weak unless the product owns:

- distribution
- school/company workflow
- saved history
- scoring
- compliance
- upload/export
- marketplace integration

## 2. Workflow Products Rose

Higher-ranked ideas have workflow resistance:

-口コミ返信自動化
-社内FAQ bot
-介護/看護記録
-建設日報
-競合価格監視
-インボイス/税務チェック

They are harder to replace because the value is not only the text.

The value includes:

- repeated work
- structured input
- saved output
- operational context
- records
- compliance or review burden
- integrations

## 3. Result Certainty Matters

High result certainty means:

```text
The user receives something usable even if the AI is not perfect.
```

Low result certainty means:

```text
The user may receive advice but cannot trust or act on it.
```

This hurts:

- SNSバズ予測
- 家計投資提案
- 農業収穫予測
- Revenue Risk Scanner

## 4. Revenue Risk Scanner Position

Revenue Risk Scanner is now:

```text
Rank 11 / 55
Total 20 / 30
Judgment: CAUTION
```

Why it is not lower:

- AI-led operation is strong
- URL input is easy
- if benchmark/history exists, generic AI resistance improves

Why it is not top:

- desire intensity is medium
- result certainty is medium
- ChatGPT/Claude can imitate a generic audit
- user may not feel pain before result

Revenue Risk Scanner should not be prioritized unless it proves:

- specific findings users cannot get from generic AI
- benchmark/history advantage
- repeated workflow
- payment intent

## Stronger Build vs Buy Patterns

A separate product becomes more buyable when it has at least two of these:

1. saved user/business context
2. recurring workflow
3. system integration
4. source data ingestion
5. rules/checklists beyond LLM
6. history and alerts
7. export or submission workflow
8. team/admin features
9. industry-specific constraints
10. trust or compliance layer

If a product has none, it is probably:

```text
just a prompt wrapper
```

and should be avoided.

## Final Recommendation

The company should not chase broad AI generation.

The best zero-base direction is:

```text
AI-powered workflow products where generic ChatGPT output is insufficient.
```

Priority categories:

1. reputation/review workflow
2. field record/report workflow
3. monitoring/alert workflow
4. compliance-adjacent checklist workflow
5. internal knowledge/FAQ workflow

Avoid:

1. generic writing
2. generic planning
3. generic copy generation
4. entertainment prompts
5. one-shot advice products
