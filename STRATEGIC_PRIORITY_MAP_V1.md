## Tier 2: Mobile First Field SaaS

### Current Status

Mobile First Field SaaS is promising but not active product development.

It remains:

```text
watchlist + paper validation candidate
```

### Priority Subareas

| Rank | Candidate | Reason |
|---:|---|---|
| 1 | Cleaning Site Report / Quote Assistant | strongest mobile-first fit, lower compliance burden |
| 2 | Repair / Maintenance Visit Report Assistant | clear recurring field workflow |
| 3 | Delivery Incident / Proof-of-Work Assistant | strong smartphone fit, but more operational complexity |

### Why It Is Interesting

- users may not use PCs heavily
- smartphone photo/voice workflows are natural
- internal build risk may be lower
- recurring field reports are common
- AI can convert messy field input into structured output

### Why It Is Not Tier 1

- no MVP yet
- no user validation yet
- customer acquisition may require field sales
- onboarding/support may be heavy
- workflow specifics differ by industry
- low PC literacy can also mean low SaaS self-service literacy

### Allowed Work

Only:

- paper validation
- interview questions
- hypothesis notes
- competitor/context notes if directly needed

### Prohibited Work

Do not:

- build a prototype
- design full product specs
- create app UI
- write code
- launch a landing page
- start broad outreach
- spend more attention than Revenue Risk Scanner

### Paper Validation Question

For cleaning/maintenance operators, ask:

```text
Do you currently create site reports, work completion reports, damage reports, or quotes from smartphone photos/voice notes?
```

Then ask:

```text
Would automatic report generation save enough time or reduce enough mistakes to pay monthly?
```

## Operating Rule

When choosing between tasks:

1. If it helps collect real Revenue Risk Scanner user feedback, do it first.
2. If it improves finding/payment/build-vs-buy validation, do it second.
3. If it only explores Mobile First, keep it lightweight.
4. If it is a new unrelated idea, defer it.
