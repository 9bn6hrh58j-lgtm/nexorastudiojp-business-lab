# Japan Readiness Audit - Sales Funnel V1

## Objective

Acquire the first paid Japan Readiness Audit customer.

This funnel is designed for customer validation, not scale. The goal is to prove that overseas small businesses will pay for a practical Japan readiness review before building the full Japan Market Entry Toolkit.

## Funnel Overview

1. Prospect acquisition
2. Sample report viewing
3. Founder Beta application
4. Payment
5. Delivery
6. Follow-up

## Funnel Principle

Do not ask prospects to buy a generic service.

Show them a specific Japan readiness problem first, then invite them to request a structured audit.

The strongest path is:

```text
Specific observation -> Sample report -> Founder Beta application -> Payment -> Audit delivery -> Follow-up offer
```

## 1. Prospect Acquisition

### Target Prospect

The initial target is a small overseas business that already has some reason to care about Japan.

Best-fit examples:

- Shopify or DTC brand with international shipping
- SaaS or digital product with global users
- Brand already shipping to Japan
- Brand with Japanese customer inquiries
- Founder-led business considering Asia expansion
- English-only website with obvious Japan trust gaps

### Primary Channel

LinkedIn outbound.

### Acquisition Motion

1. Build a list of 100 target companies.
2. Identify one decision-maker per company.
3. Review each website for one specific Japan readiness issue.
4. Send a connection request.
5. After acceptance, send a short personalized observation.

### Message Goal

The first message should not sell the audit directly.

It should create a small moment of recognition:

```text
This person noticed a real Japan-related issue on my website.
```

### Expected Drop-Off

High drop-off should be expected here.

Likely reasons:

- Japan is not a current priority.
- The recipient does not accept the connection request.
- The message feels too niche or unfamiliar.
- The prospect does not believe Japan readiness is urgent.
- The prospect wants free advice but does not want to pay.

### Drop-Off Control

Reduce drop-off by using a specific website observation:

- unclear Japan shipping
- missing customs/duties explanation
- weak return policy for overseas buyers
- no Japan-specific FAQ
- unclear payment options
- weak trust signals near checkout

## 2. Sample Report Viewing

### Purpose

The sample report proves what the prospect will receive before they pay.

It must answer:

```text
What does a Japan Readiness Audit actually look like?
```

### Required Asset

Sample audit report:

```text
japan-readiness-audit/validation-assets/SAMPLE_AUDIT_REPORT.md
```

### Recommended Customer-Facing Format

The sample report should be converted into a cleaner customer-facing format before outreach:

- PDF
- public landing page
- Notion-style page
- Gumroad free preview

GitHub source is acceptable as the project record, but it is not ideal as a customer-facing proof asset.

### CTA After Sample Report

The sample report should lead to one action:

```text
Apply for the $49 Founder Beta Audit
```

### Expected Drop-Off

Likely reasons:

- Report looks too generic.
- Prospect does not see their own situation reflected.
- Report feels like a template, not expertise.
- Sample is too long and the prospect does not finish it.
- No clear application button or payment path.

### Drop-Off Control

- Put the strongest findings near the top.
- Make the report visually clean.
- Show priority-ranked recommendations.
- Include a clear next step.
- Use the sample to sell the audit, not to explain the entire business.

## 3. Founder Beta Application

### Purpose

The application should qualify the prospect before payment.

This is important because some sites are not a fit:

- no live website
- no international selling path
- highly regulated category
- no Japan relevance
- expectation of legal/tax/import advice

### Recommended Application Fields

- Name
- Email
- Company name
- Website URL
- Country
- Product/service category
- Are you currently selling to Japan?
- Do you ship to Japan?
- Why are you considering Japan now?
- What do you want reviewed most?
- Consent that this is not legal, tax, customs, or regulatory advice

### Application Decision

Applications should be sorted into:

- Fit: send payment link
- Maybe: ask one follow-up question
- Not fit: politely decline or recommend a better next step

### Expected Drop-Off

Likely reasons:

- Too many fields.
- Prospect does not want to apply before paying.
- The offer still feels unclear.
- The prospect is not sure whether the audit is worth $49.
- The prospect expects immediate checkout instead of qualification.

### Drop-Off Control

Keep the application short.

The application should feel like:

```text
Submit your website so I can confirm if this audit is useful for you.
```

Not:

```text
Fill out a long consulting intake form.
```

## 4. Payment

### Purpose

Payment confirms real demand.

This project should not count free interest as validation.

### Founder Beta Price

```text
$49
```

This is a validation price for the first 10 companies only. It should not become the long-term price.

### Recommended Payment Tool

Gumroad is acceptable for the first version because it is fast, simple, and suitable for a digital service-style deliverable.

### Payment Flow

1. Prospect submits application.
2. Fit is confirmed.
3. Payment link is sent.
4. Prospect pays.
5. Confirmation email asks for any missing context.
6. Audit delivery clock starts after payment and required inputs are received.

### Expected Drop-Off

Payment is the most important drop-off point.

Likely reasons:

- Prospect is interested but not willing to pay.
- $49 is low enough to be affordable, but not urgent enough to purchase.
- Prospect does not trust the provider yet.
- Payment page lacks proof.
- No sample report preview.
- No clear delivery timeline.
- No clear refund/cancellation policy.

### Drop-Off Control

The payment page must include:

- what the buyer gets
- delivery timeline
- first 10 companies only
- sample report link
- who this is for
- who this is not for
- no sales guarantee disclaimer
- not legal/tax/regulatory advice disclaimer

## 5. Delivery

### Delivery Promise

Written audit report delivered in 3-5 business days after payment and intake completion.

### Delivery Contents

The audit should include:

- first impression
- key Japan readiness issues
- Japanese buyer hesitation points
- trust signal review
- FAQ gaps
- payment/shipping/return clarity review
- priority-ranked improvement list
- recommended next steps

### Delivery Format

Recommended first version:

- PDF report
- optional Markdown source in the repository for internal tracking

### Expected Drop-Off

This stage is less about drop-off and more about satisfaction risk.

Likely problems:

- Report feels too generic.
- Recommendations are not specific enough.
- Buyer expected translation or market research.
- Buyer expected guaranteed Japan sales.
- Delivery takes longer than promised.

### Drop-Off Control

- Personalize the report to the actual website.
- Include screenshots or page references when possible.
- Rank issues by priority.
- Separate "must fix before outreach" from "nice to improve."
- Restate that the audit is readiness review, not a launch guarantee.

## 6. Follow-Up

### Purpose

Follow-up turns a one-time $49 validation sale into learning, testimonials, and possible higher-value offers.

### Follow-Up Timing

1. Delivery day: send report and ask them to review.
2. 3 days after delivery: ask if recommendations were clear.
3. 7 days after delivery: ask what they plan to fix first.
4. 14 days after delivery: ask for testimonial or permission to anonymize findings.

### Follow-Up Questions

- Was the audit useful?
- Which finding surprised you most?
- Which recommendation will you implement first?
- What was missing?
- Would you pay for a deeper audit?
- Would you want implementation help?
- Can the anonymized findings be used to improve the toolkit?

### Upsell Paths

Possible next offers:

- $149 Mini Audit
- $399-$499 Full Audit
- $1,500+ Japan readiness implementation sprint
- Japan Market Entry Toolkit after patterns are validated

### Expected Drop-Off

Likely reasons:

- Buyer reads report but does not implement.
- Buyer was only curious.
- Buyer does not want follow-up.
- Buyer does not have budget for deeper help.

### Drop-Off Control

Make the follow-up specific:

```text
Which of the top 3 issues do you want to fix first?
```

Avoid vague follow-up:

```text
Let me know if you need anything.
```

## Funnel Metrics

Track these numbers weekly:

| Stage | Metric | Validation Signal |
|---|---|---|
| Prospect acquisition | Companies researched | 100 initial targets |
| Prospect acquisition | Connection acceptance rate | 20%+ is workable |
| First message | Reply rate | 10%+ is promising |
| Sample report | Sample view/request rate | 5+ requests from 100 targets |
| Application | Application submissions | 3+ from 100 targets |
| Payment | Paid audits | 1-2 from first 100 targets is early signal |
| Delivery | Useful feedback | 3+ detailed responses from first 10 audits |
| Follow-up | Upsell interest | 2+ deeper help inquiries from first 10 audits |

## Where Prospects Will Drop Off

### Highest-Risk Drop-Off Points

1. LinkedIn message ignored
2. Sample report viewed but no application
3. Application started but not submitted
4. Payment link sent but not paid
5. Audit delivered but no follow-up engagement

### Most Dangerous Drop-Off

Payment link sent but not paid.

This means the positioning created interest but not enough trust or urgency to convert.

If this happens repeatedly, do not build more assets. Fix the offer.

## Validation Criteria

The funnel is working if:

- 100 targeted companies produce at least 1 paid audit
- buyers understand the offer without a call
- payment happens before full delivery
- common objections repeat
- audit findings create follow-up interest

The funnel is weak if:

- prospects only want free advice
- Japan is rarely a near-term priority
- $49 is still not worth paying
- no one submits an application
- the sample report does not increase conversion
